### 📝 배운 내용 (Today I Learned)

- 클라이언트 - 서버 모델
```
Client - Server Model : 요청(Request), 응답(Response)의 반복으로 이루어짐
통신 방식 : 요청 -> 처리 -> 응답
장점 : 중앙 관리, 보안, 일관성
단점 : 단일 장애점(SPOF:잠재적 단일 장애 지점:한 부분의 실패가 전체 시스템의 중단을 초래하는 요소)
```
- DNS (Domain Name System)
```
인간이 식별하기 힘든 IP 주소(192.168.0.1 같은..)를 naver.com 과 같이 식별하기 쉬운 문자로 접근할 수 있음
브라우저: naver.com 입력 > DNS 서버: 도메인 IP(203.0.113.10,예시임) 반환 > 브라우저: 해당 IP로 접속
DNS의 이점 :
1. 가용성 : 장애 우회 (Server A 다운 -> Server B IP 응답)
2. 트래픽 분산 : DNS 기반 로드 분산
3. 성능 : 지리 기반 라우팅을 통한 Latency 감소
4. 보안 : IP 은닉, DDos완화
5. 운영 편의성 : 인프라 변경이 쉬움
7. 확장성 : 다중 리전, 멀티 클라우드, 글로벌 서비스 
```
- IP == IP Address == IPv4
```
IP(Internet Protocol)는 네트워크에서 장치를 식별하고 데이터가 목적지까지 전달되도록 경로를 지정하는 통신 규약
IP 주소 : 네트워크 상 장치를 구분하기 위한 논리적 주소

```
- Port
```
특정 IP 내에서 통신할 프로세스를 식별하기위한 식별자
```
- HTTP : HyperText Transfer Protocol
```
웹에서 브라우저와 서버가 데이터를 주고받기위한 프로토콜
```
- Protocol
```
웹에서 브라우저와 서버가 데이터를 주고받기위한 프로토콜
```
- HTTP Method
```
HTTP Request에 쓰이는 메서드(POST, GET, PUT, PATCH, DELETE 등)
```
- HTTP Status Code
```
클라이언트의 요청을 어떻게 처리했는지 표시해주는 코드
(200:요청 성공, 201:리소스 생성/400:잘못된요청,401:권한없음,404:not found 등등..)
```
- REST
```
URI를 통해 자원을 명시하고 HTTP Method를 통해 데이터를 처리하는 등 웹에서 클라이언트와 서버가 일관된 규칙을 따르도록 설계된 인터페이스
```
- API
```
서로 다른 소프트웨어 시스템과 통신하기 위해 따라야하는 규칙
```
- JSON
```
데이터를 저장하고 전송하기 위한 텍스트 기반의 데이터 형식 (2차원)
```
- 직렬화 — Serialization
```
복잡하게 얽힌 데이터 구조를 일렬화 하여 저장하거나 전송할 수 있는 형태로 만드는 것
```
- 역직렬화 — Deserialization
```
JSON 문자열을 파싱하여 프로그램 언어의 자료구조로 바꾸는 과정 
```
- LAN / WAN
```
LAN: 라우터 및 스위치와 같은 커넥터를 사용하여 물리적으로 가까운 디바이스를 연결하는 근거리 통신망 / WAN: 독립된 네트워크의 집합(LAN,MAN,VPC,VPN 등)을 라우팅를 통해 연결한 네트워크
```
